# Frontend

React + Vite SPA, served as static files by the backend in production.

---

## Stack

| Layer | Choice |
|---|---|
| Build tool | Vite 6 |
| UI | React 19 + TypeScript |
| Styling | TailwindCSS v4 (via `@tailwindcss/vite` — no config file needed) |
| Routing | TanStack Router (code-based, not file-based) |
| Data fetching | TanStack Query |
| Error tracking | Sentry (`@sentry/react`) |

---

## Local development

```bash
npm install
npm run dev    # http://localhost:5173
```

Vite proxies all `/api/*` requests to `http://localhost:3000`, so the backend must be running. No CORS configuration needed in development.

## Sentry setup

Sentry is initialised in `src/main.tsx` before the React tree mounts:

```ts
Sentry.init({
  dsn: "...",
  integrations: [
    Sentry.browserTracingIntegration(),  // traces page navigations and API calls
    Sentry.replayIntegration(),          // session replay on errors
  ],
  tracesSampleRate: 1.0,
  tracePropagationTargets: ["localhost", /^https:\/\/mytrainning\.up\.railway\.app\/api/],
  replaysSessionSampleRate: 0.1,
  replaysOnErrorSampleRate: 1.0,
})
```

**`browserTracingIntegration`** automatically creates spans for TanStack Router navigations and `fetch`/`XMLHttpRequest` calls to the backend, linking them to Sentry traces on the server side.

**`replayIntegration`** records session replays. 10% of sessions are sampled normally; 100% of sessions that contain an error are captured. Replays are stored in Sentry and can be viewed alongside the error event.

**`tracePropagationTargets`** limits distributed trace headers to `localhost` and the production Railway URL — this prevents Sentry from injecting trace headers into third-party requests.

The DSN is committed directly in the source because it is a public-facing identifier (not a secret). Sentry uses origin-based rate-limiting on the receiving end to prevent abuse.

## Build

```bash
npm run build    # outputs to dist/
```

In production the Dockerfile builds the frontend first (`npm run build`) then copies `dist/` into the image. The backend registers `fastify-static` pointing at `frontend/dist/` and serves `index.html` for any unmatched route (client-side routing).

## Project structure

```
src/
├── main.tsx              # app entry — Sentry init, QueryClient, Router
├── routeTree.ts          # auto-generated by TanStack Router
├── index.css             # Tailwind base styles
├── routes/               # one file per route
├── components/           # shared UI components
├── lib/                  # api client, toast, utility hooks
└── pages/                # full-page components (ProfilePage, ProgressPage, …)
```
